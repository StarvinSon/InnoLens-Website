InnoLens

A smart user management system for the Innovation Wing that analyses the collected data and provides insights into the daily operation of the facility.

What is this about?

The Innovation Wing is a place for students to conduct learning activities, experiments and showcase their achievements. It is located on the G/F and LG/F of Hui Oi Chow Science Building and is going to open in April 2020. This project is about collecting, analysing and displaying the data from the Innovation Wing to provide insights into users' usage of the facility.

Collect

Collect the data by different means such as sensors, online system and entrance gate.

Analyze

Analyze the data collected to search for any pattern or trend.

Display

Present the analysis result on a real-time dashboard located in Innovation Wing.

Project Schedule

InnoLens will be implemented and tested according to the following timeline.

Oct to Nov 2019

Prototype

In this stage, the most basic work will be done to determine the general structure of the project.

Dec to Jan 2020

Elaboration

The most challenging work will be accomplished in this stage, as to confirm whether the system is feasible.

Jan to Feb 2020

Construction (Frontend)

Most of the frontend design will be completed in this stage. The project will have a user interface for most basic functions. The corresponding backend framework will be ready as well.

Feb to Mar 2020

Construction (Backend)

Other backend functions will be implemented in this stage to finalize the system.

Mar to Apr 2020

Testing

In this stage, the system will be tested with different data, so that it can be debugged before publishing.

Oct to Nov 2019

Implement a prototype of the dashboard

A simple static webpage or paper prototype will be made to illustrate the dashboard. The group will agree on the general theme of the dashboard.

Oct to Nov 2019

Define the logic and database structure

The database structure will be defined, including the types of data and their relationship.

Dec to Jan 2020

Develop the pattern matching algorithm for hazard detecting

The model for electricity and air quality pattern matching will be developed and trained.

Jan to Feb 2020

Design staff backstage dashboard

The design agreed on previous stage will be implemented. The backstage dashboard for staff will be realized.

Jan to Feb 2020

Design user dashboard

The design agreed in previous stage will be implemented. The dashboard for users will be realized.

Feb to Mar 2020

Develop functions for normal database query operation

The analysis that requires only database operations will be implemented at the backend as well.

Mar to Apr 2020

Perform testing with mock data

Different artificial datasets, including the extreme ones will be used for testing. They are used to detect calculation errors.

Oct to Nov 2019

Perform testing with real data (from DreamLab)

Real datasets generated by the DreamLab will be used for testing. They are used to determine whether the analysis can produce useful and meaningful information.
